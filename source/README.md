# 原始素材 (Source)

这个目录用于存放项目的原始素材和资源文件。

## 目录结构建议

```
source/
├── images/           # 图片素材
│   ├── icons/        # 图标
│   ├── photos/       # 照片
│   └── illustrations/ # 插画
├── videos/           # 视频素材
├── audio/            # 音频素材
├── documents/        # 文档素材
│   ├── contracts/    # 合同文件
│   ├── licenses/     # 授权文件
│   └── templates/    # 文档模板
├── design/           # 设计源文件
│   ├── figma/        # Figma设计文件
│   ├── sketch/       # Sketch设计文件
│   └── psd/          # Photoshop源文件
└── data/             # 数据文件
    ├── csv/          # CSV数据
    ├── json/         # JSON数据
    └── sql/          # SQL脚本
```

## 文件命名规范

### 图片文件
- 使用小写字母和连字符
- 包含描述性名称
- 示例：`user-avatar-default.png`、`logo-primary.svg`

### 设计文件
- 包含版本号
- 示例：`homepage-design-v1.0.fig`、`app-icon-v2.0.sketch`

### 数据文件
- 包含日期或版本
- 示例：`users-export-2024-12-06.csv`、`products-v1.json`

## 文件格式建议

### 图片
- **图标**：SVG（矢量）或PNG（透明背景）
- **照片**：JPG（压缩）或WebP（现代格式）
- **插画**：SVG（矢量）或PNG（高分辨率）

### 视频
- **Web使用**：MP4（H.264编码）
- **高质量**：MOV或原始格式

### 音频
- **Web使用**：MP3或AAC
- **高质量**：WAV或FLAC

## 版本管理

### 大文件处理
- 使用Git LFS管理大文件
- 或使用云存储（OSS、S3等）
- 在README中记录文件位置

### 版本控制
- 重要素材保留多个版本
- 使用版本号或日期标识
- 记录修改历史

## 授权和版权

### 素材来源
记录每个素材的来源和授权信息：
- 自制素材：标注创作者和日期
- 购买素材：保存授权证明
- 免费素材：记录来源和许可协议
- 第三方素材：确保有使用权限

### 授权文件
在`documents/licenses/`目录保存：
- 图片授权证明
- 字体授权证明
- 音视频授权证明
- 第三方库授权

## 优化建议

### 图片优化
- 压缩图片大小（使用TinyPNG、ImageOptim等）
- 使用适当的分辨率（Web通常72-96 DPI）
- 考虑使用WebP格式（更小的文件大小）

### 视频优化
- 压缩视频文件
- 使用适当的分辨率和码率
- 考虑使用流媒体服务

## 注意事项

1. **版权合规**：确保所有素材都有合法使用权
2. **文件大小**：控制文件大小，避免仓库过大
3. **备份重要**：重要素材要有备份
4. **定期清理**：删除不再使用的素材
5. **文档记录**：在README中记录重要素材的信息
